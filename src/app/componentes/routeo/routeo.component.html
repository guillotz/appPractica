Routeo Component

Como ya sabemos, Angular es un framework que nos permite desarrollar páginas SPA (Single Page Application) pero para poder lograr esto, debemos generar 'rutas' que nos lleven a esos componentes que terminarán siendo la vista de nuestra página.

Para poder utilizar esas rutas, debemos configurar nuestros módulos de routeo para que Angular entienda a que componente debe llamar. El módulo de configuración de rutas inicial se llama "app-routing.module.ts".

app-routing.module.ts
  • Las rutas estan configuradas principalmente con un path y con un component
  • path: nombre de la ruta => http://localhost:4200/usuarios
  • component: componente al cual hace referencia => UsuariosComponent
Recordar importar los componentes a utilizar en el módulo
  • Una vez que tengamos las rutas configuradas, con sus path y component solo restará agregar en el decorador del módulo, la sentencia que carga esas rutas en la aplicación

Existen muchas más configuraciones para nuestras rutas, como por ejemplo la propiedad canActivate, canDeactivate, children, etc que nos permitirán configurar de una manera mucho más personalizada nuestras rutas / páginas / componentes

Documentación oficial de routeo en angular para obtener más información: Angular Router

A nivel HTML solo resta utilizar el componente <router-outlet> para poder introducir el componente relacionado en la ruta, en ese espacio

A nivel HTML no se verán los componentes que no estén en uso, sino que dentro de nuestro elemento <router-outlet> se irán cargando de manera dinamica nuestros componentes (HTML / CSS / TS)

No necesariamente tenemos que utilizar solo 1 módulo de routeo, angular nos da la posibilidad de generar todos los modulos que necesitemos para cubrir nuestra aplicación

Existen muchas maneras de poder utilizar el <router-outlet>, podemos usarlo desde el HTML o desde lógica desde nuestro TS

  • También puede ser utilizado por un botón, sin la necesidad de crear una función y capturar el evento click. esto quiere decir que podemos utilizar la propiedad [routerLink] también adentro de un botón.

  • Acceso a interpolación desde un botón

Desde nuestro archivo .TS podemos generar lógica para poder navegar entre distintos componentes de la aplicación.

  1. Se deberá importar {{ '{ Router }' }} de @angular/Router
  2. Inyectar la dependencia en el constructor del componente
  3. De la misma manera que se utiliza el [routerLink] se utilizará el metodo navigate de nuestra dependencia router

Pueden pasarse parámetros por ruta, acceder a subrutas o rutas hijas de rutas padres, por ejemplo: localhost:4200/usuarios/juan la ruta 'juan' sería una ruta hija de la ruta 'usuarios'. Para mas información revisar la documentación oficial

Ejercicios
