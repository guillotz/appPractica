Definición

La comunicación entre componentes nos sirve para intercambiar datos entre componentes, siempre existirá un componente padre que utilice a uno o más componentes hijos. Para poder realizar esta comunicación se utilizarán los decoradores @Input y @Output

Los decoradores @Input y @Output sirven para intercambiar datos entre componentes, son un mecanismo para enviar y recibir datos de un componente a otro. @Input se usa para recibir datos en un componente mientras que @Output se usa para enviar datos fuera de un componente. @Output envía datos exponiendo a los productores de eventos, generalmente objetos EventEmitter.

La finalidad del uso de estos decoradores es reutilizar los componentes, un ejemplo claro sería un componente de tipo tabla, en donde reciba la información que deba mostrar y se genere de manera dinámica permitiendo utilizar la misma tabla en toda la aplicación, solamente enviando desde el padre la información al hijo ( @Input ) y recibiendo sus eventos ( @Output )

Una de las premisas del uso de este tipo de decoradores ( @Input/@Output ) es que siempre la comunicación será de la siguiente manera:

El padre siempre se comunicará por @Input con sus hijos y los hijos se comunicarán siempre mediante @Output con su padre es una comunicación descendente de padre a hijo y ascendente de hijo a padre.

Dada la finalidad de la comunicación entre componentes, podemos encontrarnos con varios escenarios, por ejemplo un padre que tenga muchos hijos, un padre que tenga hijos que a su vez estos hijos tengan otros componentes hijos (nietos), etc. Siempre la comunicación será de manera directa entre padre e hijo, no podemos saltear 'generaciones' utilizando estos decoradores.

No existe la comunicación directa entre componentes hermanos , esto quiere decir que si un padre tiene varios hijos, el componente padre deberá ser el encargado de manejar la información entre sus hijos.
