Input

El decorador @Input como vimos en la definición nos servirá para poder enviar información de manera descendente de un componente padre a un componente hijo

Componente padre
  • El componente padre deberá llamar al componente hijo en su código .HTML, esto lo hará utilizando el nombre del selector del componente que desea emplear como hijo.
  • Una vez puesto en el su .HTML , el padre podrá enviar información al hijo utilizando [nombre_de_input_hijo] = valor
  • Ese valor podrá ser un valor fijo (como en el ejemplo: 33), podrá ser una variable del padre, el resultado de una función del padre, etc. siempre y cuando sea un valor que coincida con el tipo de dato declarado como @Input() en el componente hijo
Componente Hijo
  • El primer paso del componente hijo será importar 'Input' de @angular/core
  • Una vez importado, declarar una variable propia utilizando el decorador @Input()
  • Cuando el componente padre llame al hijo y le pase el valor de la variable, esta podrá se utilizada para cualquier fin, mostrarla por interpolación, usarla para procesar alguna lógica, etc.

Por parte del .HTML del componente hijo, solamente se muestra por interpolación el valor que recibe del componente padre

En cuanto al archivo .TS del hijo, suceden varias cosas, se importa Input de @angular/core, se define el nombre de la variable con el decorador @Input(), en este caso el nombre de la variable es propiedad y luego es la que se utiliza en el .HTML para mostrar el valor por interpolación.
